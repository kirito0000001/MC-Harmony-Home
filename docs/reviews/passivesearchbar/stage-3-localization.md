# Passive Search Bar 阶段三：汉化审查

检查对象：`Passive Search Bar 1.0.1-2026Reset` 候选。

## 玩家可见文本范围

- 源码不创建配置界面、按钮、提示、聊天消息、命令反馈或物品文本。
- JAR 不包含 `en_us.json`、`zh_cn.json` 或其他语言资源；这符合它只改变原版搜索框焦点、不新增自身 UI 的功能边界。
- `neoforge.mods.toml` 保留英文显示名称 `Passive Search Bar` 和英文静态简介。该元数据不支持按游戏语言动态切换，直接写入中文会反而破坏英文环境的兼容性。
- `pack.mcmeta` 的 `Passive Search Bar resources` 仅是资源包内部描述，不是玩家正常游玩时的功能提示。

## 双语处理结论

本阶段不修改模组原始资源：不存在需要本地化的游戏内操作文本，新增未被代码引用的语言键只会制造无效资源。

模组目录中可见的介绍、官网和问题反馈入口会在阶段四由 Codex Catalogue Bridge 提供双语覆盖：

- 英文环境显示自然英文功能说明。
- 简体中文环境显示“防止创造模式物品栏的搜索框在打开时自动抢占键盘输入焦点”等自然中文说明。
- 不改变 NeoForge 元数据、Mixin 行为或服务端边界。

## 结论与后续流程

阶段三通过，原 JAR 没有需要翻译的玩家操作文本。下一阶段是阶段四：补充 Catalogue 双语介绍、项目链接和预填 `[Passive Search Bar]` 的问题反馈入口；该阶段会扩展 Bridge 候选，不替换当前游戏文件。
