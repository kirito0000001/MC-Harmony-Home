# JER 阶段零：快速初筛

检查对象：`Just Enough Resources 1.6.0.17`

- 文件：`JustEnoughResources-NeoForge-1.21.1-1.6.0.17.jar`
- 模组 ID：`jeresources`
- 许可证：`Don't Be a Jerk` non-commercial care-free license
- 一句用途：在 JEI 中展示矿物等资源生成信息与生物掉落，帮助玩家查询物品来源。

## 初判依据

- JAR 含 149 个 class、14 种语言资源，含 `en_us` 与 `zh_cn`。
- 未发现 JER 自身的物品模型、配方数据或战利品表；它展示的是原版和其他模组已注册的资源与掉落信息。
- 元数据要求 JEI，说明它是 JEI 的信息扩展；依赖侧标为 `BOTH`，不能仅凭元数据把服务端结论定为客户端专用。

## 初步结论

- 物品等级：高置信度 `不适用`，后续只需确认没有动态物品注册即可快速通过。
- 服务端：初判 `optional_server_enhancement`，阶段二需检查是否有服务端同步、世界读取或专用网络包。
- 汉化：已有简体中文资源，阶段三核对键完整性和自然度。
- Catalogue：需要补充双语用途、官方网页和预填 `[JER]` 的问题反馈链接。

## 后续流程

下一步进入阶段一：代码与性能审查，优先检查资源/掉落扫描时机、缓存、客户端线程和对大型整合包的潜在影响。此阶段不需要关闭游戏或安装候选。
