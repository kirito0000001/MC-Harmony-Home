# Mouse Tweaks 阶段一：代码与性能审查

检查对象：`Mouse Tweaks 2.26.1`，模组 ID 为 `mousetweaks`。

## 审查范围与方法

对安装 JAR 的 NeoForge 输入事件、容器 Mixin、槽位移动算法和当前配置进行静态检查。使用 JDK 21 的 `javap` 确认实际槽位操作、事件取消范围和配置默认值。

## 核心行为

- 仅有一个客户端 Mixin：`AbstractContainerScreenAccessor`，用于访问原版容器界面的找槽位和 `slotClicked` 方法；没有向菜单、物品或网络协议注入通用逻辑。
- 鼠标按下、释放、拖拽和滚轮事件只在容器界面打开时处理。`MouseTweaksNeo` 仅在客户端初始化，非客户端环境直接返回。
- 批量移动最终调用原版容器界面的 `slotClicked`，使用 `ClickType.PICKUP` 或 `ClickType.QUICK_MOVE`。服务器仍按正常容器协议验证每个槽位操作，不存在直接篡改客户端物品栏的路径。
- 查找可推入、可拉取槽位只在鼠标事件发生时线性遍历当前容器槽位，并检查 `mayPlace`、物品兼容性、堆叠上限和结果槽位；没有每 tick、每帧或后台线程扫描。
- 当前 `MouseTweaks.cfg` 保持全部主要功能开启，且 `Debug=0`，不会为输入事件持续写调试日志。

## 风险与对策

### P2：复杂容器的输入竞争

该模组会在处理过鼠标输入后取消对应的 Screen 事件。它支持原版 `AbstractContainerScreen`，并提供 `MouseTweaksIgnore` 与 `MouseTweaksDisableWheelTweak` 注解供其他模组精确禁用功能；但极端自定义容器仍可能与拖拽、滚轮或自身快捷键竞争。

对策：当前没有崩溃或容器异常证据，保留默认设置。若某个具体容器出现双重移动、拖拽失效或滚轮冲突，先在该界面关闭滚轮操作或由对应模组适配注解，不删除 Mouse Tweaks 的其他功能。

### P3：一次滚轮操作的点击突发

滚轮和拖拽会为多个目标槽位连续调用原版点击接口。大型仓库中一次操作可能产生较多有效点击；高延迟服务器或有严格交互限速的服务端可能暂时回滚客户端预览，随后由服务器状态同步纠正。

对策：保留 `WheelTweak=1`，因为它是主要效率功能。若未来连接到有限速的服务器出现回弹或丢操作，优先将 `WheelTweak=0`，不影响左右键拖拽和快速转移；这是配置级回退，不需要改代码。

### P3：Mixin 版本耦合

Mixin 设置 `required=true` 且 `defaultRequire=1`，NeoForge 或原版 `AbstractContainerScreen` 方法签名重大变化时会在启动阶段明确失败，而不是静默失效。

对策：当前 1.21.1-NeoForge_21.1.235 已正常加载。今后升级 Minecraft、NeoForge 或 Mouse Tweaks 时，将它作为优先启动检查项；不对已验证版本修改 Mixin。

## 结论

未发现可安全合入且有实际收益的代码性能优化。槽位计算由输入事件触发，正常路径不常驻；服务器权威校验完好，当前调试日志关闭。本阶段不修改 JAR、源码或当前配置。

下一阶段是阶段二：正式确认客户端初始化、原版点击协议和无自定义 payload 后，决定是否复制到服务器模组目录。该阶段不需要关闭游戏。
