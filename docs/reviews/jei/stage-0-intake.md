# JEI 阶段零：快速初筛

检查对象：`Just Enough Items 19.38.0.366`

- 文件：`[JEI物品管理器] jei-1.21.1-neoforge-19.38.0.366.jar`
- 模组 ID：`jei`
- 许可证：MIT
- 一句用途：为物品、方块、配方和用途提供可搜索的客户端浏览界面，是整合包中其他 JEI 扩展的共同交互基础。

## 初判依据

- JAR 含 973 个 class、33 种语言资源，但没有自身物品模型、`data/jei/recipes` 或其他配方数据资源。
- 元数据的 `displayTest = IGNORE_SERVER_VERSION` 表明客户端可进入未安装 JEI 的服务器；但 NeoForge 与 Minecraft 的依赖侧均标为 `BOTH`，服务端结论仍需在源码阶段确认。
- 元数据说明明确以稳定性和性能为目标；由于它处于配方浏览与模组插件调用的中心位置，阶段一不做泛化的微优化，只检查加载、搜索索引、插件注册与客户端事件中的实际风险。

## 初步结论

- 物品等级：高置信度 `不适用`，待正式代码检查确认没有动态物品注册。
- 服务端：倾向 `client_only`，但尚未定论。
- 汉化：需要检查英文、简体中文及硬编码界面文本。
- Catalogue：需要补充中英文用途说明、官方网页和预填 `[JEI]` 的问题反馈链接。

后续阶段以此初判为依据；先完成源码与性能审查，再决定是否需要构建本地候选。
