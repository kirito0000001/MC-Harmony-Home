# JEI 阶段二：服务端部署判断

检查对象：`Just Enough Items 19.38.0.366-2026Reset`。

## 静态证据

- `META-INF/neoforge.mods.toml` 使用 `displayTest="IGNORE_SERVER_VERSION"`，明确允许装有 JEI 的客户端加入未安装 JEI 的服务器。
- 客户端的 `ConnectionToServer.canSendPacket` 会先检查服务器是否暴露 JEI 的 payload 通道；未安装 JEI 时返回 `false`，不会发送 JEI 专用数据包。
- 网络注册链调用 `.optional()`，服务端与客户端不存在 JEI 时都不会把协议不匹配当作连接阻断。
- 模组在服务端仍会注册 `SERVER` 配置与主线程网络处理。可选包覆盖物品给予、删除、热栏设置、配方转移和作弊权限查询；服务端配置只控制 OP、创造模式和 `/give` 权限下的 JEI 作弊模式。
- 客户端 GUI、渲染、搜索和物品浏览通过 `dist.isClient()` 才注册，专用服务端不加载这些界面功能。

## 结论

- 分类：`optional_server_enhancement`（可选服务端增强）。
- 普通生存服不需要安装 JEI；客户端的物品浏览、配方查询和用途查询仍可工作，也能加入未装 JEI 的服务端。
- 若希望服务端权威处理 JEI 的作弊取物、删除物品、热栏设置或 JEI 专用配方转移，可在服务端额外安装与客户端完全相同版本的 JEI。
- 当前不把 JEI 复制进 `我的世界mod服务器版本`：候选尚待 `4 / 10` 批次游戏内验证，且你的开服清单不应把可选增强误当作必装依赖。
